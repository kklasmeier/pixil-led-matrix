# Pixil Performance Metrics Viewer

**Location**: `tools/view_pixil_metrics.py`

A comprehensive tool for analyzing Pixil script performance data from the database. Provides insights into script execution times, optimization effectiveness, and JIT compilation performance.

**Run from the repository root directory.**

## Basic Usage

The tool uses subcommands. Put the command name first, then its options:

```bash
python tools/view_pixil_metrics.py <command> [options]
python tools/view_pixil_metrics.py --help
python tools/view_pixil_metrics.py trends --help
```

If you omit the command, the tool prints help (there is no default action).

## Commands

### recent — Recent Executions

```bash
# Show 10 most recent script runs (default count)
python tools/view_pixil_metrics.py recent

# Show a specific number of recent runs
python tools/view_pixil_metrics.py recent --count 20

# Recent runs for one script only
python tools/view_pixil_metrics.py recent --script "bouncing_ball.pix" --count 5
```

**Output includes**:
- Script name and execution time
- Performance metrics (commands/sec, lines/sec)
- Optimization hit rates (Fast Path, Fast Math, Cache, Parse Value)
- JIT performance (Hit Rate, Skip Efficiency) when present

### trends — Performance Trends

```bash
# System-wide trends (last 20 executions, default)
python tools/view_pixil_metrics.py trends

# Trends for a specific script
python tools/view_pixil_metrics.py trends --script "starfield.pix"

# More executions and verbose columns
python tools/view_pixil_metrics.py trends --script "fractal_trees.pix" --count 50 --verbose

# System-wide with verbose output
python tools/view_pixil_metrics.py trends --count 30 --verbose
```

Omit `--script` for system-wide trends; use `--script` for a single script.

### summary — Performance Summary

```bash
# System-wide summary (last 7 days)
python tools/view_pixil_metrics.py summary

# Summary for one script
python tools/view_pixil_metrics.py summary --script "bouncing_ball.pix"
```

### runs — Execution Runs (detailed table)

```bash
# Recent runs across all scripts
python tools/view_pixil_metrics.py runs --count 20

# Runs for one script, last 7 days, verbose columns
python tools/view_pixil_metrics.py runs --script "Boids_Flocking_Simulation.pix" --days 7 --verbose

# Only resource-efficient runs (active time ≥ 99% of total time)
python tools/view_pixil_metrics.py runs --resource-constrained --verbose
```

### jit-summary — JIT Compilation Analysis

```bash
python tools/view_pixil_metrics.py jit-summary
```

### list — Scripts in Database

```bash
python tools/view_pixil_metrics.py list
```

## Understanding the Output

### Performance Metrics

**Basic Performance**:
- **Commands/sec**: Rate of command processing
- **Lines/sec**: Rate of script line processing
- **Execution Time**: Total time including setup/teardown vs Active time (pure execution)

**Optimization Hit Rates**:
- **FP% (Fast Path)**: Simple variable lookups
- **FM% (Fast Math)**: Optimized math expressions
- **C% (Cache)**: Expression cache hit rate
- **UF% (Ultra Fast)**: Direct integers/colors/strings
- **Fast%**: Fast Path Parse (array access/simple arithmetic)
- **Cache%**: Variable Cache (LRU parameter cache)
- **ParseVal%**: Parse value cache (compact trends view only)

**JIT Compilation Metrics**:
- **JIT%**: Expressions served from JIT bytecode cache
- **Skip%**: Expressions skipped due to known failures
- **JIT-Size**: Expressions in JIT cache
- **JIT-Hit**: JIT cache utilization
- **JIT-Comp**: JIT compilation time
- **Failed**: Unique script lines that failed JIT compilation

### Trend Indicators

When viewing trends, arrows indicate performance direction:
- **↗**: Improving (faster execution, higher optimization rates)
- **↘**: Declining (slower execution, lower optimization rates)
- **→**: Stable (no significant change)

### Output Formats

**Compact format** (trends, default):

```
Date/Time           ExecTime  Cmds/s  Lines/s  FastPath%  FastMath%  Cache%  ParseVal%  JIT%  Skip%
2025-05-26 14:30    2.1s      1250    890      85%        72%        64%     58%        42%   28%
```

**Verbose format** (`--verbose` on `trends` or `runs`):

```
Date/Time           Script              TotalTime ActTime  Commands Cmds/s   Lines/s   FP%   FM%   C%  UF%  Fast% Cache% JIT%  Skip%  JIT-Size JIT-Hit  JIT-Comp Failed
```


### 3. JIT Compilation Analysis


```bash
# Overall system performance
python tools/view_pixil_metrics.py trends --count 100

# Find performance regressions
python tools/view_pixil_metrics.py trends --count 100 --verbose
```

## Interpreting Results

### Good Performance Indicators
- **High hit rates**: Fast Path >80%, Cache >60%, JIT >40%
- **Consistent performance**: Stable commands/sec across executions
- **Time savings**: Significant optimization time savings
- **Low failed lines**: Minimal JIT compilation failures

### Performance Issues to Investigate
- **Declining trends**: ↘ arrows in performance metrics
- **Low hit rates**: <50% on optimization features
- **High failed lines**: Many JIT compilation failures
- **Inconsistent performance**: Large variations in execution time

### JIT Cache Efficiency
- **Hit Rate >40%**: Good JIT utilization
- **Skip Efficiency >20%**: Effective redundant computation elimination
- **Low Failed Lines**: <10 per script indicates good expression compatibility

## Tips for Analysis

1. **Compare Similar Scripts**: Use trends to compare performance across similar animations
2. **Track Optimization Changes**: Monitor hit rates before/after code optimizations
3. **Identify Bottlenecks**: Low hit rates indicate areas for optimization
4. **Cache Effectiveness**: JIT summary shows if cache size is appropriate
5. **Performance Regression**: System trends help identify when performance declined
